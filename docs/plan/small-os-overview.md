# small OS — 概要

## プロジェクト目的・背景

**small OS** は、OS の基本的な仕組みを学習するために作られたシミュレーター型 OS です。

実際の OS カーネルを実装するには高度な環境構築が必要ですが、small OS はブラウザー上で動作するため、インストール不要・環境依存なしで OS の概念を体験的に学べます。

学習ターゲット：
- プロセス管理・スケジューリングの基礎
- ファイルシステムの概念
- シェルとコマンドインタープリターの仕組み
- メモリ管理の基礎概念

## 機能・コマンド一覧

### シェル基本コマンド

| コマンド | 説明 |
|---------|------|
| `help` | 使用可能なコマンドの一覧を表示 |
| `clear` | 画面をクリア |
| `echo <text>` | テキストを出力 |
| `exit` | セッションを終了 |

### ファイルシステム操作

| コマンド | 説明 |
|---------|------|
| `ls` | カレントディレクトリのファイル一覧 |
| `cd <dir>` | ディレクトリ移動 |
| `mkdir <dir>` | ディレクトリ作成 |
| `touch <file>` | ファイル作成 |
| `cat <file>` | ファイル内容の表示 |
| `rm <file>` | ファイル削除 |

### プロセス管理

| コマンド | 説明 |
|---------|------|
| `ps` | 実行中プロセス一覧 |
| `kill <pid>` | プロセスの終了 |

> ※ 上記は典型的な実装例です。実際にサポートするコマンドに合わせて更新してください。

## 技術スタック・アーキテクチャ

### 技術スタック

- **フロントエンド**: HTML / CSS / TypeScript（Vite でビルド）
- **実行環境**: ブラウザー（Chrome / Firefox / Safari 対応）
- **外部依存**: なし（ライブラリ不使用）

### アーキテクチャ概要

```
┌──────────────────────────────────────────────────┐
│                ユーザー空間                        │
│  ┌─────────────────────────────────────────────┐ │

│          カーネル空間    │                        │
│  ┌──────────────────────▼──────────────────────┐ │

│  │           ユーザー空間との唯一の境界          │ │
│  └───────┬──────────────┬──────────────┬───────┘ │
│          │              │              │          │
│  ┌───────▼──────┐ ┌─────▼──────┐ ┌───▼────────┐ │
│  │  プロセス管理 │ │ メモリ管理  │ │ファイル    │ │
│  │ Scheduler.ts │ │MemoryMgr   │ │システム    │ │
│  │              │ │            │ │FileSystem  │ │
│  │ PCB / ラウン │ │64KB / 4KB  │ │(VFS/inode) │ │
│  │ ドロビン     │ │ページング  │ │            │ │
│  └───────┬──────┘ └─────┬──────┘ └───┬────────┘ │
│          └──────────────┼────────────┘          │
│  ┌──────────────────────▼──────────────────────┐ │
│  │       割り込みコントローラー                  │ │
│  │         InterruptController.ts               │ │
│  │   TIMER(10ms)  /  KEYBOARD  /  SYSCALL       │ │
│  └──────────────────────┬──────────────────────┘ │
├─────────────────────────┼────────────────────────┤
│    ハードウェア抽象層    │                        │
│  ┌──────────────────────▼──────────────────────┐ │
│  │  setTimeout / setInterval / localStorage     │ │
│  │            ブラウザー API                    │ │
│  └─────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘
```

- **Terminal UI**: ユーザー入力を受け取り、出力を表示する DOM ベースの端末エミュレーター
- **Shell**: コマンドを解析し、システムコール経由でカーネル機能を呼び出す
- **Syscall（システムコール層）**: ユーザー空間とカーネルを分離する唯一の窓口
- **Scheduler**: PCB（プロセス制御ブロック）でプロセス状態を管理し、ラウンドロビンでタイムスライス切り替えを行う
- **MemoryManager**: `ArrayBuffer` 64KB をページ（4KB 単位）で管理し、実 OS のページングを模擬
- **FileSystem**: inode ベースでファイル名とデータを分離して管理
- **InterruptController**: `setInterval` でタイマー割り込みを模擬し、スケジューラーの `tick()` を定期呼び出し

## カーネル

### カーネルコンポーネント

#### システムコール層（`Syscall.ts`）

シェルやコマンドがカーネル内部を直接呼ばないよう、すべての操作を Syscall 経由に限定する。`SYS.READ`, `SYS.WRITE`, `SYS.FORK`, `SYS.EXIT` などの番号で処理を振り分け、ユーザー空間とカーネル空間の分離を模擬する。

#### プロセス管理・スケジューラー（`Scheduler.ts`）

各プロセスは PCB（Process Control Block）として管理され、`running / ready / waiting / zombie` の状態を遷移する。スケジューリングはラウンドロビン方式で、タイマー割り込みごとにタイムスライスを消費し、超過すると次のプロセスへ切り替える。

```
プロセス状態遷移:
  ready → running（スケジューラーが選択）
  running → waiting（I/O 待ち）
  running → zombie（exit 呼び出し）
  waiting → ready（I/O 完了）
```

#### メモリ管理（`MemoryManager.ts`）

`ArrayBuffer` 64KB を 4KB ページ 16 枚に分割して管理する。プロセス生成時にページを割り当て、終了時に解放する。ページテーブルで使用状況を追跡し、断片化やメモリリークの概念を体験できる。

#### 仮想ファイルシステム（`FileSystem.ts`）

inode とディレクトリエントリを分離した構造で実装する。inode はファイルのメタデータ（サイズ・タイムスタンプ・データ）を保持し、ディレクトリはファイル名と inode 番号のマッピングを持つ。この構造によりハードリンクの概念も再現できる。

#### 割り込みコントローラー（`InterruptController.ts`）

`setInterval`（10ms）でタイマー割り込みを模擬し、スケジューラーの `tick()` を定期呼び出しする。キーボード入力やシステムコールも割り込みとして扱い、実 OS の割り込み駆動モデルを再現する。

### カーネル起動シーケンス

```
1. メモリ初期化（ページテーブルのクリア）
2. ファイルシステム初期化（ルートディレクトリ作成）
3. 割り込みハンドラー登録
4. init プロセス（PID=1）生成
5. 割り込みコントローラー開始
6. シェルプロセスを fork して端末に接続
```

### 学習できる OS の概念

| カーネル機能 | 学べる概念 |
|------------|-----------|
| Scheduler + PCB | コンテキストスイッチ、プロセス状態遷移 |
| MemoryManager | ページング、断片化、メモリリーク |
| FileSystem (inode) | inode とディレクトリエントリの分離、ハードリンク |
| Syscall | ユーザー空間とカーネル空間の分離、特権モード |
| InterruptController | 割り込み駆動処理、タイマー割り込み |

## 使い方・デモ手順

### 起動方法

1. リポジトリをクローン（またはファイルをダウンロード）
2. `index.html` をブラウザーで開く
3. ターミナル画面が表示されたら操作開始

```bash
# ローカルサーバーで起動する場合
npx serve .
# → http://localhost:3000 をブラウザーで開く
```

### 基本操作デモ

```
$ help
Available commands: help, clear, echo, ls, cd, mkdir, touch, cat, rm, ps, kill

$ mkdir projects
$ cd projects
$ touch readme.txt
$ ls
readme.txt

$ echo "Hello, small OS!" > readme.txt
$ cat readme.txt
Hello, small OS!

$ ps
PID  NAME    STATUS
1    shell   running
```

### 学習シナリオ例

1. **ファイルシステム探索**: `ls` / `cd` / `mkdir` でディレクトリ構造を理解する
2. **プロセス観察**: `ps` でプロセス一覧を確認し、`kill` で終了させる
3. **シェルスクリプト**: 複数コマンドを連続実行してスクリプト的な動作を試す
